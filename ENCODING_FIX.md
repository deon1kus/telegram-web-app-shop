# Исправление проблем с кодировкой в Git

## Проблема
Старые коммиты на GitHub отображаются с неправильной кодировкой (кириллица как кракозябры типа "СЂР°Р±РѕС‡Р°СЏ РІРµСЂСЃРёСЏ").

## Решение

### 1. Настройки Git для UTF-8 (уже выполнено)

```bash
git config --global i18n.commitencoding UTF-8
git config --global i18n.logoutputencoding UTF-8
git config --global core.quotepath false
```

### 2. Обновлен .gitattributes

Файл `.gitattributes` настроен для правильной обработки текстовых файлов:
- Все текстовые файлы (*.md, *.txt, *.json, *.ts, *.tsx и т.д.) обрабатываются как UTF-8
- Бинарные файлы (изображения, шрифты) помечены как binary

### 3. Текущее состояние

✅ **Все текущие файлы** читаются нормально:
- `README.md` - корректная кодировка
- `DEPLOY_FIX.md` - корректная кодировка  
- `PRE_DEPLOY_CHECKLIST.md` - корректная кодировка
- `CHANGELOG.md` - корректная кодировка
- Все исходные файлы проекта - корректная кодировка

✅ **Последний коммит** исправлен:
- Commit `3735408` - сообщение на английском языке
- Все будущие коммиты будут с правильной кодировкой

### 4. Старые коммиты

⚠️ **Важно:** Старые коммиты в истории Git уже имеют неправильную кодировку в метаданных. Это не влияет на содержимое файлов, только на отображение сообщений коммитов на GitHub.

**Опция 1: Оставить как есть** (рекомендуется)
- Файлы проекта читаются нормально
- Будущие коммиты будут с правильной кодировкой
- Не требует переписывания истории

**Опция 2: Переписать историю** (не рекомендуется)
- Требует force push
- Может вызвать проблемы для других разработчиков
- Рискованно для публичных репозиториев

### 5. Рекомендации для будущих коммитов

1. **Используйте английский язык** для сообщений коммитов:
   ```bash
   git commit -m "fix: resolve critical useState error"
   ```

2. **Или используйте правильную кодировку** при коммитах с кириллицей:
   ```bash
   # Убедитесь, что терминал поддерживает UTF-8
   git commit -m "fix: исправлена ошибка useState"
   ```

3. **Проверяйте кодировку перед коммитом:**
   ```bash
   git log -1 --format="%B" | file -
   ```
